## Feature #253: Reduce motion setting respected

**Status:** COMPLETED ✅

**Description:** Ensure the application respects the user's prefers-reduced-motion system setting by disabling or reducing animations.

### Implementation Details

**Existing CSS Support (index.css lines 167-180):**
The application already had CSS media query support:
```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
  .grain-overlay {
    display: none;
  }
}
```

**New: Framer Motion Support (App.tsx):**
Added MotionConfig wrapper to handle Framer Motion animations:
```tsx
import { motion, MotionConfig } from 'framer-motion';

function App() {
  return (
    <MotionConfig reducedMotion="user">
      <AuthProvider>
        <BrowserRouter>
          ...
        </BrowserRouter>
      </AuthProvider>
    </MotionConfig>
  );
}
```

The `reducedMotion="user"` setting automatically detects the user's system preference and disables Framer Motion animations accordingly.
